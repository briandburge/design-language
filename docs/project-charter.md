# Project charter

## Mission

Design Language is an open specification for human and AI-assisted communication.

## Current phase

Work across two horizons:

1. Make DSL 1.x useful, testable, stable, and backward-compatible.
2. Preserve the long-term vision for a mature AI-native DSL platform.

## Working philosophy

- Optimize for usefulness.
- Avoid unnecessary abstraction.
- Prefer small, reviewable changes.
- Preserve stable rule IDs.
- Record important decisions.
- Keep the repository and Custom GPT aligned.

## Rule philosophy

Every rule should include intent, specification, rationale, preferred examples, discouraged examples, exceptions, AI notes, and related rules.

## AI philosophy

AI should amplify authorship rather than replace it.

## Engineering philosophy

Treat DSL like a mature open-source specification with semantic versioning, ADRs, RFCs, validation, reproducible behavior, and explicit release notes.
